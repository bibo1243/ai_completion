# 專案收尾與驗證計畫 (Final Verification & Delivery Plan)

目前系統已完成核心架構重構、功能實作（登入、行事曆、日記）與基礎設施（Docker、CI）。接下來將專注於品質驗證、文件製作與交付。

## 1. 測試修復與執行 (Test Fixes & Execution)
*   **修復 E2E 測試**：Playwright 設定似乎有誤 (報錯 `test() not expected`)，需檢查 `playwright.config.ts` 與測試檔案。
*   **執行完整測試套件**：確保 Unit Tests (`vitest`) 與 E2E Tests (`playwright`) 皆通過。

## 2. 性能與體驗優化 (Performance & UX)
*   **Lighthouse 檢查**：檢查應用的性能評分，優化圖片載入或程式碼分割 (Code Splitting)。
*   **路由懶加載 (Lazy Loading)**：將 `CalendarView`, `JournalView`, `Login` 改為 `React.lazy` 加載，減少首屏 bundle 大小。

## 3. 負載測試 (Load Testing)
*   **編寫 k6 測試腳本**：建立 `load-test.js`，模擬多用戶同時登入與讀取任務的場景 (目標 10,000 用戶模擬)。

## 4. 文件製作 (Documentation)
*   **README 更新**：撰寫完整的專案說明、安裝步驟、架構圖。
*   **API 文件**：簡述 Supabase Schema 與資料流。
*   **使用者手冊**：簡短的操作指引 (Markdown 格式)。

## 執行順序
1.  修復並驗證測試。
2.  實作懶加載與性能優化。
3.  建立負載測試腳本。
4.  撰寫交付文件。
